# FixAgent 调用目录树

## 整体架构

**混合架构**：
- **Orchestrator（调度中枢）** — 意图识别 + 模式路由
- **ReAct 循环** — 子 Agent 内部自主决策工具调用（Thought → Action → Observation）
- **Function Calling** — MemoryAgent 单次调用 + 向量冲突检测

```
用户输入 → Orchestrator（意图识别 + 路由分发）
  ├── CHAT → 直接 LLM 对话
  ├── RETRIEVAL → RetrievalAgent (ReAct) → 检索→评估→追问→返回
  ├── DIAGNOSIS → DiagnosisAgent (ReAct) → 检索→图谱查询→推理→诊断
  ├── GUIDANCE → GuidanceAgent (ReAct) → 检索流程→生成步骤→安全校验
  └── FULL → 串行: Retrieval → Diagnosis → Guidance → 汇总
```

## 架构关键变化（相比旧 LangChain LCEL 管道）

| 旧架构（管道式） | 新架构（ReAct + Function Calling） |
|----------------|--------------------------------|
| LCEL `RunnableBranch` 固定路由 | Orchestrator 意图识别 + LLM 自主决策 |
| 每 Chain 有固定工具调用顺序 | Agent 内部 ReAct 循环自主决定何时调什么工具 |
| LangChain `@tool` 装饰器 | 自研 `BaseTool` 基类 + `to_openai_schema()` |
| 无 trace/可观测性 | `chat_with_tools()` 返回 `trace[]`，记录每轮推理 |
| 固定输出结构 | `MemoryAgent` 用 Pydantic 校验 + `response_format={"type": "json_object"}` |
| 依赖 LangChain | 零框架依赖，直接调用 DashScope API |
| 固定 chains 层 | `IntentionType → AgentMode` 映射已内联到 `orchestrator_agent.py` |

## 异常处理架构

```
BaseTool.run()  ← 模板方法
    ├── try: _execute() → ToolResult(success=True)
    ├── except ToolException: → ToolResult(success=False, error=ToolError)
    └── except Exception: → ToolResult(success=False, code="TOOL_ERROR")

BaseAgent.run()  ← 模板方法
    └── except Exception: → AgentOutput(status="error", error_type, error_detail)

BaseAgent.run_with_react()  ← ReAct 模式
    ├── except RuntimeError: → "推理步骤超出限制" (max_iterations exceeded)
    └── except Exception: → AgentOutput(status="error")

api/main.py 端点
    ├── 检测 metadata.status="error" → HTTPException(500)
    └── 全局异常捕获 → JSONResponse(status_code=500)
```
